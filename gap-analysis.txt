SECURITY FIX — Mass Assignment Vulnerability in Guest Update Endpoint

## Context (verified against code)
`PUT /bookings/:id/guests/:guestId` (bookings/router.ts line 277-298) passes raw `req.body` directly to Prisma `update()` at line 285: `data: req.body`. This allows an attacker to override ANY field — including `tenantId`, `bookingId`, `cFormSubmitted`, etc.

Note: The `POST /:id/guests` (line 275) extracts fields explicitly (`fullName`, `nationality`, `idProofNumber`, `visaNumber`, `visaExpiryDate`, `purposeOfVisit`, `arrivingFrom`, `goingTo`) — so the POST is NOT directly vulnerable, but it still lacks Zod validation.

## Task
1. **Audit** every route in `packages/api/src/modules/bookings/router.ts` that uses `req.body` directly with Prisma. The confirmed vulnerable line is **285** (`data: req.body`). Search for other instances across all modules: `grep -rn "data: req.body" packages/api/src/modules/`
2. **Create a Zod schema** called `bookingGuestSchema` in `packages/shared/src/validators/index.ts` (where ALL existing Zod schemas live — 14 schemas, 228 lines). The schema must match the actual `BookingGuest` model fields: `fullName`, `nationality`, `idProofNumber`, `visaNumber`, `visaExpiryDate` (optional date string), `purposeOfVisit`, `arrivingFrom`, `goingTo`.
3. **Validate** `req.body` against this schema. The codebase uses BOTH `.parse()` (most validators) and `.safeParse()` (night-audit/router.ts line 21). Pick one pattern and be consistent — `.safeParse()` with explicit error handling is recommended for better UX.
4. **Fix the PUT endpoint** at line 285 to use the validated+extracted fields instead of raw `req.body`.
5. **Apply the same pattern** to every other route that passes `req.body` directly to Prisma. Search: `grep -rn "data: req.body" packages/api/src/modules/`
6. **Write/extend Playwright E2E test** in `04-frontdesk-operations.spec.ts` (3.3KB) that attempts to set `tenantId` or `bookingId` via the guest update endpoint and verifies it's rejected or ignored.

## Acceptance Criteria
- [ ] Zero routes in `packages/api/` pass raw `req.body` to Prisma
- [ ] Zod schema added to `packages/shared/src/validators/index.ts`
- [ ] Schema field names match actual Prisma model (fullName, NOT firstName/lastName)
- [ ] E2E test proves tenant override attempt fails
- [ ] No `any` types introduced (per INSTRUCTIONS.md Domain #2)
- [ ] Confidence: 8/10
